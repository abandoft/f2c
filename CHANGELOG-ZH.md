## 1.7.0

- 将 `PRINT` 迁移到统一 token、AST 和类型化 I/O 流程，移除文本切割与表达式二次解析。
- 统一 `PRINT`、`READ` 和 `WRITE` 的格式化传输引擎及格式语义诊断。
- 支持 `PRINT` 的列表导向、字符常量、运行时字符、标签格式、空输出列表和嵌套 implied-DO。
- 支持 F77 已赋值 FORMAT 变量，并生成动态、无固定容量的格式标签选择逻辑。
- 修正格式末尾定位描述符产生多余记录空格的问题，使输出记录与原生 Fortran 一致。
- 增加严格 C17、ASan/UBSan、原生 Fortran 输出差分及固定 Reference BLAS/LAPACK 回归。

## 1.6.0

- 将宏定义管理拆分为独立、有界的预处理模块，并支持源码内函数式宏。
- 增加零参数、嵌套和可变参数宏，以及字符串化、token 粘贴和宏重定义检查。
- 增加反斜杠续接的预处理指令逻辑行及宏展开生成的 include operand。
- 完善 `#if` 的 64 位有符号/无符号整数、字符常量、溢出和非法运算诊断。
- 增加宏参数资源限制，并扩充严格 C17、sanitizer 和固定 Reference BLAS/LAPACK 回归。

## 1.5.0

- 将外部文件控制语句迁移到统一 token、AST、类型化 IR 和结构化代码生成流程。
- 完善 `OPEN` 与 `CLOSE` 的文件状态、访问方式、动作、格式、位置、字符控制属性及文件处置语义。
- 新增顺序文件 `BACKSPACE` 与基于标准 C17 的物理 `ENDFILE` 截断实现。
- 新增按文件单元或文件名执行的 `INQUIRE`，返回连接状态、能力和文件属性。
- 统一文件控制语句的 `IOSTAT`、`IOMSG` 和 `ERR` 错误处理，并增加生成前语义检查。
- 增加严格 C17、sanitizer、执行回归、gfortran 差分和数值验证流水线覆盖。

## 1.4.0

- 将 `DATA` 语句完整迁移到统一 token、AST 和类型化 IR 流程，支持多组数据、重复因子和任意嵌套隐式 DO。
- 增加初始化常量、类型与 kind、元素数量、数组边界、重复初始化及资源限制的生成前语义检查。
- 修正标量、数组、字符、复数及 `EQUIVALENCE` 根存储的一次性初始化和 `SAVE` 生命周期语义。
- 将可移植的标量、整数组、部分数组及隐式 DO 初始化生成为高效、可读的 C17 静态初始化器。
- 扩充严格 C17、重复调用、负向诊断和固定 Reference BLAS/LAPACK 官方数值差分覆盖。

## 1.3.0

- 新增独立、有界的条件预处理器，支持动态条件嵌套、完整整数表达式求值和递归对象宏展开。
- 增加 `#include`、Fortran `INCLUDE`、`#line` 和数字行标记，并通过回调保持核心库不依赖文件系统。
- 增加 API 预处理定义以及 CLI `-D`、`-U`、`-I` 选项，明确项目级与源码局部宏作用域。
- 统一预处理、源码规范化与 token 的源码映射，结构化诊断可同时报告宏展开位置和定义位置。
- 增加预处理字节、宏数量与深度、include 数量与深度等请求级资源限制及负向测试。
- 扩充独立预处理、CLI include、跨编译器、sanitizer 和固定 Reference LAPACK 全量回归。
- 修复严格 LLVM/Linux 构建中预处理测试的 `SIZE_MAX` 声明可移植性问题。
