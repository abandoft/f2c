# F2C 实现状态与路线图

本文档记录经过验证的真实实现状态。只有具备自动化回归测试的项目才能标记为
`[x]`；“部分完成”的项目仍保持 `[ ]`，直到其中列出的要求全部实现并纳入 CI 门禁。

## 项目与构建系统

- [x] 使用可移植的 C17 实现转译器及生成代码所需的辅助功能。
- [x] 使用 CMake 构建库、命令行程序和测试，所有构建产物均位于根目录的 `build/` 中。
- [x] 按编译器职责将实现拆分为 `cli`、`core`、`frontend`、`semantic`、`ast`、`codegen` 和
      `internal` 子目录，以路径替代 `statement_`/`expression_` 等重复文件名前缀。
- [x] 在 CI 使用的 GCC、Clang 和 MSVC 配置下无警告构建项目自身代码。
- [x] 在 Linux、macOS 和 Windows 上运行 Debug/Release 原生构建与测试。
- [x] 使用 Emscripten 构建 WebAssembly 版本的转译器。
- [x] 在 CI 中运行 AddressSanitizer 和 UndefinedBehaviorSanitizer。
- [x] 将 GitHub Actions 按跨平台构建与测试、sanitizer、生成结果可复现性、WebAssembly、
      BLAS/LAPACK 数值正确性、性能、覆盖引导模糊测试和发布拆分为独立工作流；每条流水线均
      设置最小权限、并发取消、明确超时和专属诊断产物，所有缓存与输出只写入 `build/`。
- [x] 增加贯穿源码归一化、解析、语义分析、AST 和代码生成的确定性变异模糊测试，并在
      sanitizer CI 中持续执行 2,400 个自由/固定格式变异样本。
- [x] 增加 Clang libFuzzer 覆盖引导目标和每周 sanitizer 作业；版本化种子与跨作业演进语料库

      自动归档去重。
- [x] 增加多输入项目 C17 实现与共享接口头的重复转译逐字节确定性门禁。

- [ ] 增加跨构建主机的可复现二进制检查与版本化生成代码快照。

- [ ] 支持预处理器行映射，并在每条诊断中保留文件、行和列信息。
- [ ] 覆盖 Hollerith 常量以及 LAPACK 工具代码使用的全部旧式固定格式词法边界。

- [x] 在 AST 节点上记录基础 Fortran 类型、rank 和可定义性，并使用这些信息生成调用临时量。
- [x] 为 AST 增加独立单元测试，覆盖整数幂、无空格点运算符、外部函数和带步长数组切片。
- [x] 将数值类型提升集中到独立类型模块，将 intrinsic 识别与结果类型解析集中到语义注册表；
      表达式降级递归携带类型，不再从已生成的 C 文本反向恢复数值类型。
- [x] 将表达式节点所有权、词法分析、优先级解析、语义类型和 C17 输出拆分为独立模块；
      固定 Reference LAPACK 的 3,535 个文件全部使用 AST 唯一生产路径，旧字符串解析器已删除。
- [x] 将生产路径中的表达式解析为具有明确所有权的类型化 AST，不再使用旧字符串表达式
      解析器直接拼接 C 文本。
- [x] 显式表示字面量、名称、单目/双目运算、调用、数组引用、数组切片、数组构造器、子串、
      复数构造和关键字实参。
- [x] 为数组构造器增加递归隐式 `DO` AST，覆盖一个循环中的多个值、任意嵌套、动态上下界、
      缺省/正/负步长以及外层迭代变量在内层边界和值表达式中的作用域。数值和定长字符整数组
      构造器赋值完全从 AST 生成，先用带溢出/分配/精确元素数检查的 libc 堆存储物化，再一次性
      写回以保证别名安全；已知常量形状、非整数迭代器/边界、零步长、类型冲突及构造器外的
      隐式 `DO` 均产生硬错误。严格 C17 执行测试与 gfortran 逐字节差分覆盖动态、嵌套、
      负步长、内嵌构造器和字符数组元素，并保持全部固定 Reference 门禁通过。
- [ ] 为每个 AST 节点补齐 kind、完整 shape 和值类别；当前符号和表达式均具有独立 kind、
      最大 rank-15 的逐维 shape 及常量/变量/临时量/过程值类别。显式、假定大小、假定形状和
      延迟形状已区分；常量上下界、数组构造器、数组段及 elemental 运算会保留已知 extent，
      赋值、二元数组运算、elemental intrinsic 和过程实参会在生成前拒绝可证明的不合形。
      rank-1 整数向量下标现已进入 selector AST、shape 传播和别名安全的临时量写回路径；
      `SUM`/`PRODUCT`/`MINVAL`/`MAXVAL`/`MINLOC`/`MAXLOC`/`COUNT`/`ANY`/`ALL`/
      `DOT_PRODUCT` 已具备 rank-1 连续或带步长数组视图，`TRANSPOSE` 与 rank-2 `MATMUL`
      具备严格 C17 执行后端。`RESHAPE`/`PACK`/`UNPACK`/`SPREAD`/`CSHIFT`/`EOSHIFT`/
      `FINDLOC` 现已具有独立后端，覆盖 `ORDER`/`PAD`/`VECTOR`/`FIELD`、运行时 `DIM`、标量或
      数组 `SHIFT`/`BOUNDARY`、`MASK` 和 `BACK`，并通过严格 C17 与原生 Fortran 执行回归。
      字符结果现按元素长度执行截断/空格填充，字符 `FINDLOC` 使用 Fortran 比较；派生类型结果

- [ ] 补齐字符拼接临时量和所有 kind 组合；六种字符关系运算、数值类型提升、整数幂、复数与
